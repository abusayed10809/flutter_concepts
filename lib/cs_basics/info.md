Object-Oriented Programming (OOP) and Functional Programming

# Object-Oriented Programming (OOP) Subsections

- Classes and Objects: Instances, attributes/fields, methods, and constructors.
- The Four Pillars of OOP: Encapsulation, Abstraction, Inheritance, and Polymorphism
- Access Modifiers: Public, private, protected, and default/package-private control.
- Interfaces and Abstract Classes: Contracts, abstract methods, and multiple inheritance
  alternatives.
- Composition vs. Inheritance: "Has-a" relationships versus "is-a" relationships.
- Coupling and Cohesion: Designing modules that are highly cohesive and loosely coupled.

# Functional Programming (FP) Subsections

- First-Class and Higher-Order Functions: Treating functions as values, passing functions as
  arguments, and returning functions.
- Pure Functions and Side Effects: Deterministic outputs and avoiding external state modification.
- Immutability: Persistent data structures and preventing data mutation.
- Function Composition: Combining multiple simple functions to build complex ones.
- Closures: Functions that capture lexical scope variables from outside their block.
- Lambda Expressions: Anonymous functions and concise syntax for inline operations.
- Currying and Partial Application: Transforming a function with multiple arguments into a sequence
  of single-argument functions.
